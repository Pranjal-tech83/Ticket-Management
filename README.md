⚡ High-Performance API  •  🧠 AI Agent Integration  •  🗄️ Relational Database
--- > 💡 **SupportPilot Backend** is a robust, enterprise-ready API layer built with FastAPI and SQLAlchemy. It powers the SupportPilot AI Ticket Resolution Agent, handling everything from user management and ticket state to Knowledge Base search, AI response generation, escalations, and live dashboard analytics. ---
🚀 1. Setup & Installation
Get the local development environment running in seconds. ```bash # 1. Navigate to the backend directory cd supportpilot-backend # 2. Install dependencies pip install -r requirements.txt # 3. Setup environment variables (Defaults to local SQLite if skipped) cp .env.example .env ``` ---
🌱 2. Load Demo Data (Recommended)
Populate your database instantly so the frontend and AI teams can start building against real data right away. ```bash python seed_data.py ``` **What this generates:** * `3` Users * `4` Tickets (in various states like open, pending, resolved) * `3` Knowledge Base Articles * `2` AI Responses * `1` Escalation * `2` Linked Jira Tickets ---
⚡ 3. Run the Server
Spin up the FastAPI server with live-reloading enabled. ```bash uvicorn app.main:app --reload ``` | Resource | Local URL | Description | |:---|:---|:---| | **API Base URL** | `http://127.0.0.1:8000` | The root of your backend server | | **Swagger UI** | `http://127.0.0.1:8000/docs` | Interactive API documentation (Test endpoints live!) | | **ReDoc** | `http://127.0.0.1:8000/redoc` | Alternative, clean API documentation | ---
🗄️ 4. Switching to Postgres (Production/Staging)
By default, the app uses a local SQLite file (`supportpilot.db`) for zero-friction setup. When you are ready for staging or production, simply update your `.env` file. SQLAlchemy handles the dialect translation automatically—no code changes required! ```env # Inside your .env file DATABASE_URL=postgresql://user:password@localhost:5432/supportpilot ``` ---

🤖 7. Notes for the AI Agent Team
Two specific integration points are intentionally left as simple stand-ins so the API contract remains stable while the heavy AI logic gets built: * **Vector Search Preparation:** `crud.search_kb` currently uses a `LIKE` keyword match. You can safely replace its internals with FAISS/Chroma vector search later. The `/api/knowledge-base/search` endpoint signature won't need to change. * **Classification Webhook:** Call `PATCH /api/tickets/{id}/classification` once your classification engine has scored a ticket. It accepts any subset of `category`, `sub_category`, `priority`, `severity`, `classification_confidence`, and `status`. ---
🎨 8. Notes for the Frontend Team
* **Pagination:** Every list endpoint supports `skip` and `limit` pagination natively. * **CORS:** Cross-Origin Resource Sharing is currently wide open (`allow_origins=["*"]`) for frictionless local development. Please notify the backend team before pushing to production so we can lock it down to your actual frontend origin. * **Timezones:** All timestamps are standardized as **UTC ISO-8601 strings**. Ensure your frontend localized these for the user.
